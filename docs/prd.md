# เอกสารความต้องการ

## 1. ภาพรวมแอปพลิเคชัน

### 1.1 ชื่อแอปพลิเคชัน
ระบบจัดคิวและคำนวณเวลาทำงาน (GTA V FiveM Queue & Time Tracking)

### 1.2 คำอธิบาย
ระบบจัดการคิวและติดตามเวลาทำงานสำหรับ GTA V FiveM โดยใช้ Supabase เป็นฐานข้อมูล Realtime, Vercel สำหรับ Hosting และ GitHub (https://github.com/Damviplaos/Medic-gtav-op) สำหรับ Version Control พร้อม CI/CD Auto-deploy ผ่าน Vercel เมื่อ Push ไป GitHub และใช้ Supabase Realtime Subscriptions สำหรับอัปเดตข้อมูลแบบ Real-time

## 2. ผู้ใช้และสถานการณ์การใช้งาน

### 2.1 ผู้ใช้เป้าหมาย
- **Super Admin**: บัญชีหลักสำหรับสร้าง User คนอื่น
- **Admin**: สามารถสร้าง Username/Password ให้ User, จัดการยศ, ตั้งค่าห้อง, ดูรายงานภาพรวม
- **User**: ผู้ใช้ทั่วไปที่เข้าคิว, ย้ายห้อง, ขึ้นเป็น OP, ดูแดชบอร์ดตัวเอง

### 2.2 สถานการณ์การใช้งานหลัก
- Super Admin สร้างบัญชี Admin และ User
- Admin สร้างบัญชีให้ User, จัดการยศ, กำหนดเกณฑ์เลื่อนยศ, ตั้งค่าห้องที่นับเวลา
- User เข้าสู่ระบบ, เข้าคิวในห้อง「พร้อมทำงาน」, ย้ายห้องตามสถานะ, ขึ้นเป็น OP, ตรวจสอบเวลาทำงานและเกณฑ์เลื่อนยศ
- Admin/ยศสูงดูรายงานภาพรวมทุกคน

## 3. โครงสร้างหน้าและฟังก์ชัน

### 3.1 โครงสร้างหน้าหลัก
```
├── หน้า Login
├── หน้ารอคิว (หน้าแรกหลัง Login)
│   ├── แสดงรายชื่อออนไลน์
│   ├── ห้องย่อย 4 ห้อง
│   ├── กล่อง「คนรับ OP」
│   └── ระบบคิวและลูกศรมือชี้
├── หน้า Settings
│   ├── เปลี่ยนรหัสผ่าน
│   ├── ตั้ง Nickname
│   └── ตั้งชื่อในเกม (IC Name)
├── หน้า Dashboard (User)
│   ├── สรุปเวลาทำงาน
│   ├── รายสัปดาห์
│   └── ปฏิทินดูย้อนหลัง
├── หน้า Dashboard (Admin/ยศสูง)
│   └── ตารางภาพรวมทุกคน
└── หน้าจัดการยศ (Admin)
    ├── สร้างยศ
    ├── ตั้งชื่อและสีประจำยศ
    ├── ลากจัดลำดับ Hierarchy
    └── กำหนดเกณฑ์เลื่อนยศ
```

### 3.2 หน้า Login
- ผู้ใช้กรอก Username และ Password
- ตรวจสอบข้อมูลและเข้าสู่ระบบ
- เมื่อ Login สำเร็จ ระบบพาไปหน้ารอคิวและเพิ่มไอดีเข้าห้อง「พร้อมทำงาน」ทันที

### 3.3 หน้ารอคิว (หน้าแรกหลัง Login)

#### 3.3.1 แสดงรายชื่อออนไลน์
- แสดงรายชื่อทุกคนที่ออนไลน์พร้อมจุดสีเขียว (Online Indicator)
- แสดง Nickname และ IC Name

#### 3.3.2 ห้องย่อย 4 ห้อง
- **[พร้อมทำงาน]**: ห้องรอคิวหลัก
- **[เหม่อ]**: AFK
- **[ไปกิจกรรม]**: ไปทำภารกิจชั่วคราว
- **[ออกเวร]**: ออกงานแต่ยังออนไลน์อยู่
- แต่ละห้องแสดงรายชื่อคนในห้องนั้น
- ปุ่ม「เมนู >」ท้ายชื่อแต่ละคน เปิด dropdown เลือกย้ายห้อง

#### 3.3.3 กล่อง「คนรับ OP」
- แสดงด้านบนสุดของหน้า
- แสดงรายชื่อคนที่อยู่ในสถานะ OP
- ปุ่ม「สุ่ม」สุ่มดึงคนจากห้อง「พร้อมทำงาน」ขึ้นเป็น OP
- ปุ่ม「ถัดไป」(สีเขียว มุมขวาบน) เฉพาะคนที่อยู่ใน OP เท่านั้นเห็น

#### 3.3.4 ระบบคิวและลูกศรมือชี้
- แสดงคิวในห้อง「พร้อมทำงาน」เรียงตามเวลาที่เข้ามา (First In First Out)
- รูปมือชี้ 👉 ระบุคิวปัจจุบัน
- เมื่อคนในคิวปัจจุบันกดปุ่ม「ถัดไป」ลูกศรมือชี้วนไปคนถัดไปอัตโนมัติ
- ปุ่ม「ขึ้นเป็น OP」/ 「เลิกเป็น OP」สำหรับ User สลับสถานะ OP

### 3.4 หน้า Settings
- **เปลี่ยนรหัสผ่าน**: User กรอกรหัสผ่านเก่าและรหัสผ่านใหม่
- **ตั้ง Nickname**: User กรอก Nickname
- **ตั้งชื่อในเกม (IC Name)**: User กรอก IC Name

### 3.5 หน้า Dashboard (User)
- **สรุปเวลาทำงาน**: แสดงเวลาทำงานรวมวันนี้, อาทิตย์นี้, เวลา OP
- **รายสัปดาห์**: แสดงเวลาทำงานแต่ละวันในสัปดาห์
- **ปฏิทินดูย้อนหลัง**: เลือกวันดูเวลาทำงานย้อนหลัง
- **สถานะผ่านเกณฑ์**: แสดงข้อความสีเขียว「คุณผ่านเกณฑ์แล้ว กรุณาติดต่อยศสูงกว่าเพื่อสอบเลื่อนขั้น」เมื่อผ่านเกณฑ์

### 3.6 หน้า Dashboard (Admin/ยศสูง)
- แสดงตารางภาพรวมทุกคน
- คอลัมน์: ชื่อ, วันนี้ออนกี่ชั่วโมง, อาทิตย์นี้รวมกี่ชั่วโมง, OP กี่ชั่วโมง, สถานะผ่านเกณฑ์

### 3.7 หน้าจัดการยศ (Admin)
- **สร้างยศ**: Admin กรอกชื่อยศ, เลือกสีประจำยศ
- **ลากจัดลำดับ Hierarchy**: Admin ลากยศเพื่อจัดลำดับ
- **กำหนดเกณฑ์เลื่อนยศ**: Admin ตั้งค่า
  - เกณฑ์ชั่วโมงออนทำงานรวมขั้นต่ำ/สัปดาห์
  - เกณฑ์ชั่วโมงทำหน้าที่ OP ขั้นต่ำ/สัปดาห์
- **มอบยศให้ User**: Admin เลือก User และยศที่ต้องการมอบ (User 1 ไอดี ถือหลายยศพร้อมกันได้)

### 3.8 หน้าจัดการบัญชี (Super Admin/Admin)
- **Super Admin**: สร้างบัญชี Admin และ User
- **Admin**: สร้างบัญชี User โดยกำหนด Username และ Password

### 3.9 หน้าตั้งค่าห้อง (Admin)
- Admin เลือกห้องที่ต้องการตั้งค่า
- เลือกว่าห้องนั้นนับเวลาหรือไม่นับเวลา

## 4. กฎธุรกิจและตัวตรรกะ

### 4.1 ระบบบัญชีและการเข้าสู่ระบบ
- Super Admin สร้างบัญชี Admin และ User
- Admin สร้าง Username/Password ให้ User
- User เข้าสู่ระบบด้วย Username/Password
- เมื่อ Login สำเร็จ ระบบเพิ่มไอดีเข้าห้อง「พร้อมทำงาน」ทันที

### 4.2 ระบบห้องและการย้ายห้อง
- User สามารถย้ายระหว่างห้อง 4 ห้องได้ตลอดเวลา
- การย้ายห้องทำผ่านปุ่ม「เมนู >」ท้ายชื่อ
- ระบบอัปเดตสถานะห้องแบบ Real-time ผ่าน Supabase Realtime Subscriptions

### 4.3 ระบบคิวและ OP
- คิวในห้อง「พร้อมทำงาน」เรียงตาม First In First Out
- User กดปุ่ม「ขึ้นเป็น OP」เพื่อเข้าสถานะ OP
- ปุ่ม「สุ่ม」สุ่มดึงคนจากห้อง「พร้อมทำงาน」ขึ้นเป็น OP
- เฉพาะคนที่อยู่ใน OP เท่านั้นเห็นปุ่ม「ถัดไป」
- เมื่อกดปุ่ม「ถัดไป」ลูกศรมือชี้วนไปคนถัดไปในคิว
- เมื่อลูกศรมือชี้ถึงคนสุดท้ายในคิว ลูกศรวนกลับไปคนแรกอัตโนมัติ

### 4.4 ระบบคำนวณเวลาทำงาน
- Admin ตั้งค่าว่าห้องไหนนับเวลา/ไม่นับเวลา
- เมื่อ User อยู่ในห้องที่นับเวลา ระบบนับเวลาสะสมต่อเนื่อง
- เมื่อ User ย้ายระหว่างห้องที่นับเวลา เวลาไม่รีเซ็ต
- เมื่อ User เข้าห้องที่ไม่นับเวลา (เช่น ออกเวร) ระบบหยุดนับเวลา
- เมื่อ User กลับเข้าห้องที่นับเวลา ระบบนับเวลาต่อจากเดิม
- ระบบนับเวลา OP แยกต่างหากเมื่อ User อยู่ในสถานะ OP

### 4.5 ระบบยศ
- Admin สร้างยศ, ตั้งชื่อ, สีประจำยศ, ลากจัดลำดับ Hierarchy
- User 1 ไอดี ถือหลายยศพร้อมกันได้
- Admin กำหนดเกณฑ์เลื่อนยศ:
  - เกณฑ์ชั่วโมงออนทำงานรวมขั้นต่ำ/สัปดาห์
  - เกณฑ์ชั่วโมงทำหน้าที่ OP ขั้นต่ำ/สัปดาห์
- ระบบตรวจสอบอัตโนมัติว่า User ผ่านเกณฑ์หรือไม่
- เมื่อ User ผ่านเกณฑ์ ระบบแสดงข้อความสีเขียว「คุณผ่านเกณฑ์แล้ว กรุณาติดต่อยศสูงกว่าเพื่อสอบเลื่อนขั้น」

### 4.6 ระบบรายงาน
- Admin/ยศสูงเห็นตารางภาพรวมทุกคน
- User ทั่วไปเห็นเฉพาะแดชบอร์ดตัวเอง
- ระบบคำนวณเวลาทำงานรวมวันนี้, อาทิตย์นี้, เวลา OP
- ระบบแสดงสถานะผ่านเกณฑ์ในตารางภาพรวม

## 5. สถานการณ์ผิดปกติและกรณีขอบเขต

| สถานการณ์ | การจัดการ |
|-----------|----------|
| User พยายาม Login ด้วย Username/Password ผิด | แสดงข้อความแจ้งเตือน「Username หรือ Password ไม่ถูกต้อง」|
| User พยายามเปลี่ยนรหัสผ่านโดยกรอกรหัสผ่านเก่าผิด | แสดงข้อความแจ้งเตือน「รหัสผ่านเก่าไม่ถูกต้อง」|
| User ที่ไม่ใช่ OP พยายามกดปุ่ม「ถัดไป」| ปุ่มไม่แสดงให้ User ที่ไม่ใช่ OP |
| Admin ลบยศที่ User ถืออยู่ | ระบบลบยศออกจาก User อัตโนมัติ |
| User ออกจากระบบ (Logout/Offline) | ระบบหยุดนับเวลา, ลบชื่อออกจากรายชื่อออนไลน์ |
| ห้อง「พร้อมทำงาน」ไม่มีคนเมื่อกดปุ่ม「สุ่ม」| แสดงข้อความแจ้งเตือน「ไม่มีคนในห้องพร้อมทำงาน」|
| User ย้ายห้องขณะที่ลูกศรมือชี้อยู่ที่ตัวเอง | ลูกศรมือชี้วนไปคนถัดไปในคิวอัตโนมัติ |

## 6. เกณฑ์การตรวจรับ

1. Super Admin สร้างบัญชี Admin สำเร็จ
2. Admin สร้างบัญชี User พร้อม Username/Password สำเร็จ
3. User เข้าสู่ระบบด้วย Username/Password และระบบพาไปหน้ารอคิวพร้อมเพิ่มไอดีเข้าห้อง「พร้อมทำงาน」ทันที
4. User ย้ายระหว่างห้อง 4 ห้องผ่านปุ่ม「เมนู >」สำเร็จ และระบบอัปเดตสถานะแบบ Real-time
5. User กดปุ่ม「ขึ้นเป็น OP」และระบบแสดงชื่อใน กล่อง「คนรับ OP」
6. คนที่อยู่ใน OP กดปุ่ม「ถัดไป」และลูกศรมือชี้วนไปคนถัดไปในคิวอัตโนมัติ
7. Admin ตั้งค่าห้องที่นับเวลา/ไม่นับเวลา และระบบนับเวลาทำงานสะสมต่อเนื่องเมื่อ User อยู่ในห้องที่นับเวลา
8. Admin สร้างยศ, กำหนดเกณฑ์เลื่อนยศ และระบบตรวจสอบอัตโนมัติแสดงข้อความ「คุณผ่านเกณฑ์แล้ว กรุณาติดต่อยศสูงกว่าเพื่อสอบเลื่อนขั้น」เมื่อ User ผ่านเกณฑ์
9. Admin/ยศสูงเปิดหน้า Dashboard และเห็นตารางภาพรวมทุกคนพร้อมข้อมูลเวลาทำงานและสถานะผ่านเกณฑ์
10. User เปิดหน้า Dashboard ตัวเองและเห็นสรุปเวลาทำงาน, รายสัปดาห์, ปฏิทินดูย้อนหลัง

## 7. ฟังก์ชันที่ไม่ดำเนินการในเวอร์ชันนี้

- การแจ้งเตือนผ่าน Discord, LINE หรือช่องทางอื่น
- การส่งออกรายงานเป็นไฟล์ Excel/PDF
- การตั้งค่าเวลาทำงานขั้นต่ำต่อวัน
- การจัดการวันหยุด/ลา
- การบันทึกประวัติการเปลี่ยนยศ
- การแสดงกราฟสถิติเวลาทำงาน
- การตั้งค่าสิทธิ์แบบละเอียด (Fine-grained Permissions)
- การสำรองข้อมูลอัตโนมัติ
- การรองรับหลายภาษา
- การแสดงประวัติการย้ายห้อง
